Dynamic CGI Execution

Execute server-side scripts with full environment variable support

WebSrv executes server-side scripts through the Common Gateway Interface. The server handles environment variable setup, request body passing, and response processing. Scripts can be written in Python, Perl, Bash, and other languages.

Example Request

POST /cgi-bin/script.py HTTP/1.1
Host: localhost:8080
Content-Type: application/x-www-form-urlencoded
Content-Length: 19

name=John&age=30

CGI Script (Python Example)

#!/usr/bin/env python3
import os
import sys

print("Content-Type: application/json")
print()

name = os.environ.get('QUERY_STRING', '')
print(f'{{"greeting": "Hello {name}"}}')

Server Response

HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: 29

{"greeting": "Hello John&age=30"}

Key Features

  • Full HTTP environment variable setup (REQUEST_METHOD, PATH_INFO, QUERY_STRING, etc.)
  • Request body passed via stdin to script
  • Support for multiple interpreters (Python, Perl, Bash, etc.)
  • Configurable per file extension
  • EOF handling for chunked requests
  • Proper process management with fork()
  • Script execution in correct directory
  • Response header and body processing
