Cookie Management

HTTP cookie handling for stateful client-server interactions

WebSrv implements full cookie support for maintaining user state across requests. The server parses Cookie headers from client requests and sends Set-Cookie headers in responses, enabling session management and preference persistence.

Browser sends cookie

GET /cookies.html HTTP/1.1
Host: localhost:8080
Cookie: theme_color=#ff6b35

Server sets cookie

HTTP/1.1 200 OK
Set-Cookie: theme_color=#ff6b35; Path=/
Content-Type: text/html

<html><body>Color saved!</body></html>

Implementation

  • Parse incoming Cookie header from HTTP requests
  • Extract cookie name-value pairs separated by semicolons
  • Send Set-Cookie header in HTTP responses
  • Support cookie attributes: Path, Expires, HttpOnly, Secure
  • Handle form submissions that save user preferences as cookies
  • Retrieve and apply saved cookies on subsequent page loads
  • Clear cookies by setting expiration to past date
  • Cookies persist across page reloads and browser sessions
  • Browser automatically includes cookies in every request to matching domain/path

Source

See inc/CookieHandler.hpp and src/CookieHandler.cpp for the C++ implementation of cookie parsing, setting, and clearing.
