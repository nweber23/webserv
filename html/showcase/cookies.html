Cookie Management

Server-side cookie handling for stateful interactions

WebSrv implements full cookie support for maintaining user state across requests. The server parses Cookie headers from client requests and sends Set-Cookie headers in responses, enabling session management and user preference persistence.

Server Sets Cookie in Response

HTTP/1.1 200 OK
Set-Cookie: session_id=abc123xyz; Path=/
Set-Cookie: user=john; Path=/; Expires=Wed, 21 Oct 2025 07:28:00 GMT
Content-Type: text/html

<html><body>Welcome back!</body></html>

Client Sends Cookies in Request

GET /dashboard HTTP/1.1
Host: localhost:8080
Cookie: session_id=abc123xyz; user=john

C++ Server Parsing Cookies

// Parse Cookie header from request
std::string cookieHeader = request.headers["Cookie"];

// Split by semicolon and parse key=value pairs
std::map<std::string, std::string> cookies;
std::istringstream iss(cookieHeader);
std::string pair;

while (std::getline(iss, pair, ';')) {
    size_t pos = pair.find('=');
    if (pos != std::string::npos) {
        std::string key = pair.substr(0, pos);
        std::string value = pair.substr(pos + 1);
        cookies[key] = value;
    }
}

C++ Server Setting Cookies

// Build Set-Cookie response headers
HttpResponse response;
response.status = 200;
response.headers["Set-Cookie"] =
    "session_id=xyz789; Path=/; HttpOnly";
response.headers["Content-Type"] = "text/html";
response.body = "<html>...</html>";

Key Features

  • Parse incoming Cookie headers from client requests
  • Send Set-Cookie headers in server responses
  • Support cookie attributes (Path, Domain, Expires, HttpOnly, Secure)
  • Session state management across multiple requests
  • User authentication and authorization flows
  • Persistent user preferences and settings
  • Cookie expiration and lifetime management
  • Secure stateful client-server interactions
